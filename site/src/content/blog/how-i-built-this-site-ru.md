Я собрал этот сайт за одну сессию. От пустой папки до продакшн-деплоя на Cloudflare Workers. Рассказываю как и почему именно так.

## Задача

Мне нужны были две вещи, которые обычно не уживаются вместе:

1. **Красивый личный сайт** — проекты, блог, контакты. Анимации, скорость, чистый дизайн.
2. **База знаний** — digital garden. Граф связей, бэклинки, поиск. Markdown-first.

Обычно выбирают один фреймворк и заставляют его делать оба дела. Это всегда плохо. Блог-движок — плохая вики. Вики — плохой лендинг.

## Решение: два движка, один домен

Разделил:

- **Astro** — главный сайт (`/`, `/projects`, `/blog`, `/contact`). Компоненты, статика, быстро.
- **Quartz** — база знаний (`/kb/`). Создан для этого — граф, вики-ссылки, поиск, всё из коробки.

Оба генерируют статический HTML. Оба складывают в одну папку `dist/`. Cloudflare Workers раздаёт с одного домена.

```
Astro build  →  dist/         (основной сайт)
Quartz build →  dist/kb/      (база знаний)
merge.mjs    →  склейка sitemap
wrangler     →  деплой
```

Никаких конфликтов роутинга. Никакого общего рантайма. Никаких компромиссов.

## Стек

- **Astro 5** — генератор статики + Tailwind CSS 4
- **Quartz 4.5** — движок базы знаний (Preact, MDX, граф)
- **Cloudflare Workers** — хостинг на edge, режим статических ассетов
- **pnpm** — монорепо
- **@fontsource** — локальные шрифты Inter + JetBrains Mono (без Google Fonts, без блокировки рендера)

## Что потребовало больше всего мысли

### Quartz на `/kb/` — резолвинг путей

Quartz генерирует все ссылки как относительные URL. Настройка `baseUrl: "hyrahmo.com/kb"` делает sitemap и RSS корректными. Билд идёт сразу в `dist/kb/`. Никакой перезаписи путей.

### Синхронизация темы между Astro и Quartz

Оба используют `localStorage.getItem('theme')`. Astro ставит `data-theme` на `<html>`. Quartz ставит `saved-theme`. Ключ в localStorage один — переключение темы на главном сайте сохраняется в KB.

Чтобы не было белой вспышки при входе в KB в тёмной теме, я поставил `saved-theme="dark"` как дефолт в HTML-шаблоне Quartz.

### Склейка sitemap

Astro генерирует `sitemap-index.xml`. Quartz генерирует `kb/sitemap.xml`. Скрипт `merge.mjs` на 15 строк патчит индекс Astro — добавляет ссылку на sitemap Quartz.

### Мультиязычность

8 языков в KB (EN, RU, ZH, ES, DE, FR, PT, AR). Главный сайт — EN и RU с правильными `hreflang` тегами и dropdown-переключателем. Каждый язык на сайте ведёт в соответствующий раздел KB.

## Что бы сделал иначе

1. **Начать с контента, а не дизайна.** Слишком долго полировал анимации и цвета, прежде чем было что показать. Сайт выглядел дорого, но был пуст.
2. **Меньше заглушек в KB.** 185 пустых категорий на 8 языках — бессмысленно. 10 настоящих статей > 185 заглушек.
3. **Компоненты с первого дня.** Начал с одного файла layout на 270 строк. Пришлось рефакторить в Header/Footer/SocialIcons потом.

## Пайплайн сборки

```bash
npm run build
# 1. rm -rf dist
# 2. cd kb && npm install
# 3. pnpm --filter @hyrahmo/site run build  →  dist/
# 4. cd kb && npx quartz build -o ../dist/kb  →  dist/kb/
# 5. node scripts/merge.mjs  →  патч sitemap
```

Одна команда. Чистый выход. Деплой через `npx wrangler deploy`.

## Цифры

- 28 страниц Astro (EN + RU)
- 185 страниц KB (8 языков)
- Время сборки: ~4 секунды
- Lighthouse: 94+ accessibility, локальные шрифты, нет сторонних зависимостей
- Коммитов за первую сессию: 30+
